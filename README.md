# Simon-says-game
<br>Simon says game has four colored buttons, each blinking when it is pressed or activated by the device.<br>
<br>🎮 Simon Says Game<br>
A simple and interactive browser-based game inspired by the classic Simon Says memory challenge. This project was built as part of my learning journey in HTML, CSS, and JavaScript.

<br>Purpose:-<br>
The goal of the game is to test the user’s memory. At each level, the game highlights a sequence of colored buttons in a particular order. The player must repeat the exact sequence by clicking the buttons in the correct order. The sequence grows by one step every level.<br>

This project helped me practice:<br>
DOM manipulation<br>
Event handling<br>
Arrays and sequences<br>
Game logic<br><br>

Features:-<br>
Press any key to start the game<br>
Randomly generated color sequence that increases in length each level<br>
Visual button flash animations<br>
Real-time input validation<br>
Score tracking and game-over message<br>
Reset and replay functionality<br><br>

Technologies Used :-<br>
HTML: To structure the game layout (title, instructions, buttons).<br>
CSS: For layout styling, button coloring, and visual effects.<br>
JavaScript: To manage game logic, track sequences, handle user input, and provide feedback.<br><br>

What I’ve Learned:-<br>
As a beginner, I’ve learned the following by building this project:<br>

How to listen for keyboard and click events<br>
Use of arrays to store and compare sequences<br>
How to manipulate the DOM dynamically<br>
Implementing delays using setTimeout<br>
The importance of clear variable naming and code readability<br>
Basics of modularizing logic using functions (levelUp(), flash(), correct(), etc.)<br>
